# Google AI Studio Output Review Checklist

## Review purpose

Use this checklist to safely review the first web-app prototype generated by Google AI Studio from the [Google AI Studio First Prompt](google-ai-studio-first-prompt.md) and [Prototype Requirements](prototype-requirements.md).

The goal is to review the generated prototype before treating it as usable application code. The first review should focus on UI completeness, basic behavior, state handling, security, and future Chatbase integration readiness.

Do not move the generated prototype toward production until this checklist is complete and follow-up engineering review has happened.

## UI checklist

Verify that the generated prototype includes the expected visible UI areas:

- [ ] **Welcome area**: The app has a clear title or introduction explaining the prototype purpose.
- [ ] **Chat input**: The app has a visible input field and submit/send control.
- [ ] **Conversation display**: The app shows a readable message history area.
- [ ] **Loading state**: The app has a visible loading indicator or pending state.
- [ ] **Empty state**: The app shows a useful empty state before the first message is sent.
- [ ] **Error state**: The app includes a visible error or fallback area.
- [ ] **Simple form input**: The app includes a small form example with basic fields.
- [ ] **Placeholder action/API result area**: The app shows where a future action or API result would appear.
- [ ] **Responsive layout**: The app remains usable on desktop and mobile-sized screens.

## Behavior checklist

Verify that the generated prototype demonstrates the minimum interaction loop:

- [ ] Message input works.
- [ ] User message appears in the conversation display after submit.
- [ ] Loading state appears after sending a message.
- [ ] Placeholder assistant response appears after the loading state.
- [ ] Error state can be simulated or clearly previewed.
- [ ] Form input can be submitted, previewed, or otherwise demonstrated with placeholder behavior.

## Security checklist

Confirm the generated prototype does not include production or sensitive information:

- [ ] No production API keys are present.
- [ ] No production secrets are present.
- [ ] No real webhook URLs are present.
- [ ] No personal data is hardcoded.
- [ ] No unsafe external scripts are included unless clearly justified and reviewed.

If any item above fails, do not share or deploy the generated prototype until the issue is removed.

## Chatbase integration readiness checklist

Review whether the generated prototype can later be connected to Chatbase safely and cleanly:

- [ ] API layer is isolated behind a clearly named function, module, placeholder, or integration boundary.
- [ ] Placeholder response can later be replaced with a Chatbase-related backend/API response.
- [ ] Error handling is visible to the user and not only logged to the console.
- [ ] Streaming support is not required yet, but the future path for adding streaming is noted or easy to identify.

## Out of scope for first review

Do not require the first generated prototype to include:

- Production deployment.
- Real authentication.
- Payment.
- Analytics dashboard.
- Multi-agent management.

These items should be handled in later product, security, and engineering planning passes.

## Pass / fail criteria

### Pass

The prototype passes the first review if:

- All required UI areas are present or intentionally represented as placeholders.
- The basic message flow works with placeholder behavior.
- Loading, empty, and error states are visible.
- The simple form can be demonstrated.
- No production API keys, production secrets, real webhook URLs, or hardcoded personal data are present.
- Future Chatbase integration points are clear enough for the next engineering pass.

### Fail

The prototype fails the first review if:

- It includes any production API key, production secret, real webhook URL, or hardcoded personal data.
- It cannot demonstrate the basic message input and response flow.
- It lacks visible loading and error states.
- It mixes placeholder frontend code with production-like credentials or deployment assumptions.
- It is too complex to review as a first prototype.

## Next steps after review

After completing the checklist:

1. Record pass/fail notes and any missing UI or behavior items.
2. Remove any accidental sensitive data immediately.
3. Decide whether to revise the Google AI Studio prompt or manually adjust the prototype.
4. Identify the next integration target, such as placeholder API replacement, Chatbase backend/API connection, or streaming response support.
5. Keep production deployment, authentication, payment, analytics dashboard, and multi-agent management out of scope until a later review phase.
